Java vs Javax vs Jakarta:
- Java is the core programming language and platform that provides the basic libraries and APIs for developing applications. It is the foundation upon which other Java technologies are built.


- Javax is a package that contains additional libraries and APIs that are not part of the core Java platform. It includes various extensions and APIs for specific functionalities, such as Javax.servlet for web development and Javax.persistence for database access.
 

- Jakarta is the new name for the Java EE (Enterprise Edition) platform. It is a set of specifications and APIs for building enterprise applications in Java. Jakarta EE provides additional features and capabilities for developing large-scale, distributed, and multi-tiered applications, such as support for web services, servlets, and enterprise JavaBeans (EJBs). 

The name change from Java EE to Jakarta EE was made to reflect the transition of the platform from Oracle to the Eclipse Foundation, which now oversees its development and maintenance.
